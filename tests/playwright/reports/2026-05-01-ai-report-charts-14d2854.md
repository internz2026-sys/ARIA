## Diagnosis

### FAIL 1 — AI Report charts not rendered as `<img>` (the primary regression)
Root cause: `frontend/app/(dashboard)/campaigns/[id]/page.tsx` uses a hand-rolled `Markdown` component. Its `renderInline()` function (line 47) only handles `**bold**`, `*italic*`, and `` `code` `` — no `![alt](url)` image pattern. When the backend's `analyze_report` generates `ai_report_text` containing `![April Ad Performance Funnel](https://....png)`, the renderer treats it as a plain paragraph and emits the literal string as `<p>` text. Fix: add an image pattern to `renderInline()` that converts `![alt](url)` to `<img src={url} alt={alt} className="w-full rounded-lg my-3" />`, or replace the Markdown component with `react-markdown` + `remark-gfm` which handles image nodes natively.

### FAIL 2 — Inbox brief still contains chart images (regression from commit 14d2854)
The inbox item tested ("Facebook Ad Campaign: SMAPS-SIS for Philippine Parochial Schools", Ad Strategist, 34m ago) was written before the test run, so it was generated by the pre-14d2854 agent code path. However, to confirm whether 14d2854 actually removed charts from NEW briefs, a fresh delegation would need to be triggered. The 4 chart `<img>` tags visible in the inbox brief (all from `/content/charts/` in Supabase) indicate that either: (a) this particular inbox item pre-dates the commit, or (b) the chart-suppression logic in the Ad Strategist skill / server-side inbox creation path is not working. Since the item is labeled "34m ago" and the commit was earlier, this is likely a pre-existing item; a fresh delegation test is needed to confirm 14d2854 is effective for new briefs.
