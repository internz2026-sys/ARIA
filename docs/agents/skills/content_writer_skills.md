# Content Writer Skills

## SEO Blog Post Template

**Optimal Word Counts:**
| Type | Words |
|------|-------|
| Standard blog | 1,200-2,000 |
| Pillar content | 2,000-3,000 |
| Landing page | 500-800 |

**Blog Skeleton:**
```
H1: [Primary Keyword in first 4 words] + [Benefit] (60 chars max)
Meta: 155-160 chars, keyword + value proposition

TL;DR summary paragraph (goes at TOP)
Table of Contents (for posts > 1,000 words)

H2: Address primary query directly
  - Topic sentence answering the H2
  - 2-3 supporting paragraphs
  - 1 list or visual element

H2: Deep-dive section
  H3: Subsection with examples/data

H2: FAQ Section (target featured snippets)
  H3: Question format headers

H2: Conclusion + CTA
```

**Keyword Rules:**
- Primary keyword: 7-12 times per 750-1,000 words
- Place in: title, first paragraph, 2+ H2 headings
- Front-load keywords in first 4 words of H1

## Headline Formulas

1. `[Number] [Adjective] Ways to [Desired Outcome]`
2. `How to [Action] That [Specific Result]`
3. `Are You [Making This Common Mistake]?`
4. `Why [Common Belief] Is Wrong (And What to Do Instead)`
5. `How to [Desired Outcome] Without [Pain Point]`
6. `[Result] in [Timeframe]: [Method]`
7. `Stop [Bad Action]. Start [Good Action] Instead.`
8. `The Complete Guide to [Topic] in [Year]`
9. `What Every [Role] Needs to Know About [Topic]`
10. `Warning: [Thing They're Doing] Is [Negative Consequence]`

**Rules:** Under 60 chars, odd numbers outperform, use power words (proven, essential, secret, effortless).

## Content Scoring (0-100)

| Criteria | Weight |
|----------|--------|
| Search Intent Match | 20% |
| Readability (Flesch 60-70) | 15% |
| Structure & Scannability | 15% |
| Depth & Completeness | 15% |
| Keyword Optimization | 10% |
| Originality & Value | 10% |
| CTA & Conversion | 5% |
| E-E-A-T Signals | 5% |
| Technical SEO | 5% |

90+: Publish immediately. 75-89: Minor edits. 60-74: Revisions needed. <60: Rewrite.

## Product Hunt Launch Copy

**Tagline (max 60 chars):**
- Start with verb: "Automate...", "Build...", "Ship..."
- Templates: `[Verb] your [thing] in [timeframe]` | `[Outcome] without [pain]` | `[Tool type] for [audience]`

**Description (max 500 chars):**
1. What it does + who it's for
2. Key differentiator
3. 2-3 bullet features
4. Social proof

## Show HN Copy

**Title:** `Show HN: [Name] – [Plain description]`
- Zero marketing speak, direct factual language
- First-person voice: "I built...", "My side project..."
- Best posting: Tuesday-Thursday

**Body (100-200 words):**
- Problem you had → What you built → Tech stack → What's live → Feedback wanted

## E-E-A-T Signals

Always include: Author byline, experience markers ("In our testing..."), external citations, last updated date.
